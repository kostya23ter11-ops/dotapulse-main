## Fix 1: IP Validation for Rate Limiting
**File:** `app/api/ai/route.ts` (line 129)

**Problem:** `x-forwarded-for` header can be spoofed by clients, allowing rate limit bypass.

**Fix:** On Vercel, `req.ip` is the canonical way to get the real client IP (Vercel strips/proxies `x-forwarded-for`). Use `req.ip` as primary, fallback to `x-forwarded-for` only in dev.

```ts
// Before
const ip = req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() || 'unknown';

// After
const ip = req.ip || req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() || 'unknown';
```

## Fix 2: Strengthen Auth Cookie Security
**File:** `app/api/auth/callback/route.ts` (line 88)

**Problem:** `sameSite: 'lax'` allows cookie to be sent on cross-site GET navigations. `strict` is safer for auth tokens since OpenID callback is always same-origin.

**Fix:**
```ts
// Before
sameSite: 'lax',

// After
sameSite: 'strict',
```

## Fix 3: Sanitize Error Responses
**Files:** `app/api/ai/route.ts`, `app/components/ErrorBoundary/ErrorBoundary.tsx`

**Problem:** Server errors return raw error messages to the client, potentially leaking internals.

### 3a. AI Route (`app/api/ai/route.ts` line 237)
```ts
// Before
return Response.json({ error: 'Ошибка сервера' }, { status: 500 });

// After — already safe, no change needed here (generic message). 
// But line 44 leaks data.error:
// Before
throw new Error(data.error || 'Something went wrong');
// After
throw new Error('Something went wrong');
```

### 3b. ErrorBoundary (`app/components/ErrorBoundary/ErrorBoundary.tsx` line 46)
```tsx
// Before
{error?.message || t('error.message')}

// After — only show error.message in development
{process.env.NODE_ENV === 'development' ? error?.message : t('error.message')}
```

## Fix 5: Add Missing Security Headers
**File:** `vercel.json`

**Problem:** Missing CSP, Permissions-Policy, and X-Frame-Options could be stronger.

**Fix:** Add `Content-Security-Policy` and `Permissions-Policy` headers:
```json
{ "key": "Content-Security-Policy", "value": "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' https://cdn.cloudflare.steamstatic.com https://avatars.steamstatic.com data:; font-src 'self'; connect-src 'self'; frame-ancestors 'none';" },
{ "key": "Permissions-Policy", "value": "camera=(), microphone=(), geolocation=()" }
```
